# SOMA_AI_COLLECTOR_SPEC.md - Спецификация «Сборщика» (The Collector)

## 🎯 Цель модуля
Превратить «грязный» захват документа (фото/PDF) в структурированные медицинские данные (JSON $\to$ SwiftData) с минимальным участием пользователя.

## 🛠 Технический флоу (4 слоя)

### 1. Слой захвата (Local)
- **Инструмент**: Apple Vision Framework.
- **Действие**: Извлечение сырого текста (OCR) локально на устройстве.
- **Цель**: Получить «текстовый слепок» страницы без отправки изображения в облако.

### 2. Слой фильтрации и подготовки (Local)
- **Действие**: Локальный анализ текста (Soma-Local).
- **Проверки**: 
    - Язык документа.
    - Валидация: действительно ли это мед. документ.
    - Выделение ключевых зон (результаты, референсы).
- **Анонимизация (CRITICAL)**: Удаление PII (ФИО, телефоны, адреса) перед отправкой.

### 3. Слой структурирования (Cloud)
- **Инструмент**: Soma Brain (LLM: GPT-4o/Claude 3.5).
- **Процесс**: Преобразование сырого текста в JSON по схеме `LabMarker`.
- **Требования**: 
    - Жесткий маппинг на коды LOINC.
    - Флаг `needs_review: true` при низком доверии к данным.
    - Запрет на постановку диагнозов (фильтр стоп-слов).

### 4. Слой валидации и сохранения (Local)
- **Human-in-the-loop**: Экран верификации, где пользователь подтверждает правильность распознанных данных.
- **Сохранение**: Запись в SwiftData (`LabTest` $\to$ `LabMarker`).

## 🛡️ Compliance & Security (Вердикт Совета Критики)
- **Apple Store**: Никаких «диагнозов» $\to$ использование формулировок «возможные причины».
- **Privacy**: Механизм «Доверия» $\to$ возможность авто-анонимизации, но при `SENSITIVE_HIGH` — обязательное ручное подтверждение.
- **Validity**: Использование `labStandardId` для нормализации данных между разными лабораториями.
